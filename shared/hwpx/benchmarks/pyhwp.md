# Benchmark Note: pyhwp

## 왜 봤나

`pyhwp`는 HWP binary v5 parser/processor다. HWPX와 포맷은 다르지만, 같은 한글 문서 생태계 문제를 다룬다.

특히 보고 싶은 점은 셋이다.

- 분석/추출 중심 도구를 어떻게 노출하는가
- CLI와 라이브러리 경계를 얼마나 분명히 두는가
- 실험적 변환 기능을 어디까지 공식 표면으로 취급하는가

## 관찰

### 1. 분석 중심 표면이 아주 선명하다

`hwp5proc`는 `version`, `header`, `summaryinfo`, `ls`, `cat`, `unpack`, `records`, `models`, `find`, `xml` 같은 서브커맨드를 앞에 세운다.

핵심 인상:
- 편집보다 먼저 분석이 온다.
- 사용자는 파일을 열자마자 무엇을 볼 수 있는지 바로 안다.
- 읽기 전용 작업 단위가 이름만으로 이해된다.

HWPX 스택에 주는 시사점:
- `python-hwpx`와 `hwpx-mcp-server`도 inspection 계열 진입점을 더 선명하게 드러낼 수 있다.
- 특히 구조 조회, 텍스트 추출, 검증, 패키지 내부 보기 같은 읽기 전용 흐름은 더 명시적인 작업 단위로 보이는 편이 좋다.

### 2. 변환 기능은 실험적 경계가 분명하다

ODT/HTML/TXT 변환은 문서에서 `_Experimental_`로 계속 표시된다.

핵심 인상:
- 가능한 일과 안정된 일을 일부러 구분한다.
- 기능을 숨기지 않지만, 안정성 기대치를 과장하지도 않는다.

HWPX 스택에 주는 시사점:
- `hwpx-mcp-server`나 `hwpx-skill`에서 고급/실험적 워크플로가 있다면, 기본 성공 경로와 같은 톤으로 섞지 않는 편이 낫다.
- preview 성격, 제한적 지원, edge-case 많은 기능은 더 노골적으로 경계를 쳐야 한다.

### 3. CLI 문서가 작업 단위 중심이다

pyhwp 문서는 기능 소개보다 `어떤 명령으로 무엇을 본다`가 분명하다. 게다가 hacking guide, changes, 한국어 문서까지 이어진다.

핵심 인상:
- 사용자가 문제를 푸는 흐름으로 문서가 조직돼 있다.
- 도메인 사용자를 위한 한국어 문서 접근성도 인상적이다.

HWPX 스택에 주는 시사점:
- `python-hwpx` 문서도 API 객체 설명만이 아니라 `헤더 확인`, `본문 추출`, `표 구조 보기`, `검증 실행` 같은 작업명 기준 링크를 더 강화할 수 있다.
- `hwpx-skill` 트리거 문구도 `추출`, `구조 확인`, `검증`, `치환`처럼 사용자가 떠올리는 작업 단위로 더 정리할 수 있다.

### 4. 도메인 인접성이 주는 현실감이 있다

pyhwp는 한국어 문서 포맷을 다룬다는 점 자체가 중요하다.

핵심 인상:
- 범용 문서 라이브러리에서 놓치기 쉬운 사용 맥락을 보정해 준다.
- 사용자가 기대하는 질문, 확인 방식, 진단 흐름이 더 현실적이다.

HWPX 스택에 주는 시사점:
- `python-hwpx`가 단순 편집기 API가 아니라 한국어 문서 자동화/분석 도구라는 메시지를 더 분명히 해도 된다.
- `pyhwpx` 같은 COM 자동화 계열과 비교할 때도, `python-hwpx`는 포맷 직접 조작과 분석에 강하다는 포지셔닝을 더 선명히 할 수 있다.

## 바로 흡수할 액션

### python-hwpx
- docs 인덱스에 `문서 구조 보기`, `텍스트 추출`, `검증`, `패키지 심화` 같은 inspection 중심 바로가기를 더 강화한다.
- 편집 예제뿐 아니라 읽기/분석 예제를 더 전면 배치한다.

### hwpx-mcp-server
- 읽기 전용 구조 조회 흐름과 즉시 저장 편집 흐름을 더 명확히 분리한다.
- 응답 설계도 `무엇을 보여주는 도구인지`가 한눈에 드러나게 유지한다.

### hwpx-skill
- 설치 후 최소 성공 경로를 `추출 -> 확인 -> 치환 -> 재검수`처럼 분석-수정-검수 루프로 계속 고정한다.
- 트리거 문구를 작업명 중심으로 더 다듬는다.

### 공통 스택
- 실험적/제한적 기능은 validation report, README, skill 문서에서 같은 표현으로 표기한다.

## 한 줄 결론

**pyhwp에서 배울 핵심은 한글 문서 도메인에서 분석 중심 도구 표면을 얼마나 명확하게 보여주고, 실험적 기능의 경계를 얼마나 솔직하게 긋는가다.**
